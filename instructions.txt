Simulação: Teste (... e Qualidade de Software)
>>> Código comentado, com identificadores significativos e type hint
>>> Código documentado (DocString. PEP 257)
>>> Código limpo e organizado em módulos (PEP 8)
>>> Evidência de utilização de Revisão de Código - pode ser informal (pull request)
>>> Tratamento de exceções com uso de exceções UDT (User-Defined Data Type)
>>> DocTest para unidades simples
>>> Testes unitários utilizando técnicas de caixa-preta
>>> Evidência de utilização de TDD - para cada integrante, duas unidades devem ser 
desenvolvidas com TDD (RED-GREEN: utilizar comentários para deixar cada estado do 
código. Commit antes e depois do REFACTOR)
>>> Um relatório em PDF com a cobertura do código deve ser fornecido na página da aplicação 
no GitHub
